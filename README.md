# نسخة محسّنة — مؤسسة شادي الملكاوي

هذه النسخة راجعت التكامل بين الطلبات، المستودع، المشتريات، المرتجعات، التحويلات، التقارير، CRM، والصلاحيات.

## أهم التحسينات

- تحديث تصميم لوحة القيادة إلى واجهة تشغيلية حديثة مع ملخص فوري لمعدل التسليم، الإيراد، المخزون، ومخاطر المتابعة.
- تحسين دعم الموبايل بإضافة صفحات الإدارة المهمة للقائمة السفلية وجعلها قابلة للتمرير.
- إصلاح منطق خصم المخزون عند إنشاء الطلب أو إضافة صنف أو تغيير حالة الطلب.
- منع الخصم/الإرجاع الخاطئ عند وجود أكثر من صنف بنفس المقاس واللون داخل نفس العملية.
- إصلاح منطق المرتجعات حتى لا يتم إرجاع المخزون مرتين عند حذف الطلب أو إلغاء الطلب لاحقاً.
- إصلاح تعديل كمية الطلب ليقوم بتسوية فرق المخزون بدقة بدلاً من تعديل الطلب فقط.
- إصلاح التحديث الجماعي للحالات والحذف الجماعي مع تسوية المخزون بشكل موحّد.
- منع صرف مخزون أعلى من الرصيد المتوفر في التحويلات والعمليات الجماعية بالباركود.
- تحسين تصدير Excel ليعرض الأصناف المتعددة داخل الطلب الواحد، إجمالي الكمية، وحالة خصم المخزون.
- تحويل شعارات GitHub الخارجية إلى ملف محلي SHM.png لضمان ظهور الشعار عند الرفع أو العمل دون اعتماد على رابط خارجي.
- تحسين الأمان الأساسي: حفظ الحسابات لا يخزن كلمة المرور محلياً، وكلمات المرور الأساسية أصبحت hashes بدلاً من نص واضح.
- رفع إصدار Service Worker لتجنب بقاء ملفات قديمة في الكاش بعد النشر.

## ملاحظات مهمة قبل النشر

- ارفع كل الملفات كما هي، واجعل `index.html` في جذر المشروع.
- بعد النشر على GitHub/Cloudflare، افتح الموقع مرة ثم اعمل hard refresh إذا ظهرت نسخة قديمة بسبب الكاش.
- مفاتيح Firebase في تطبيقات الويب ليست سرّية بالكامل بطبيعتها، لكن يجب ضبط Firebase Rules بعناية لأن الحماية الحقيقية تكون من قواعد قاعدة البيانات والصلاحيات.
- لم أغير أسماء مسارات Firebase الحالية حتى لا تنقطع البيانات الموجودة.
