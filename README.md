# مؤسسة شادي الملكاوي — نظام إدارة المبيعات والمستودع

نظام ويب تشغيلي لإدارة الطلبات، التوصيل، المخزون، المشتريات، المرتجعات، التحويلات، العملاء، المستخدمين، التقارير، وسجل التدقيق. يعمل كتطبيق صفحة واحدة `Single Page App` باستخدام HTML/CSS/JavaScript مع Firebase Realtime Database.

---

## 1. ملخص النسخة

**الإصدار:** `2026.06.17-fab-remember-readme`  
**نوع المشروع:** Static Web App  
**قاعدة البيانات:** Firebase Realtime Database  
**نقطة التشغيل:** `index.html`  
**ملف المنطق الرئيسي:** `app.js`  
**ملفات التصميم:** `style.css` + `style-shm.css`

هذه النسخة تضيف وتصحح التالي:

- إصلاح تباين زر الإضافة العائم `FAB` ليظهر بوضوح على الخلفيات الفاتحة والداكنة.
- تحسين زر `FAB` لإضافة طلب/توصيل جديد أثناء التمرير.
- إظهار زر صغير لإرجاع شريط التنقل عند إخفائه أثناء التمرير.
- إصلاح ميزة تذكّر كلمة المرور والحسابات المحفوظة.
- تحسين تباين الحسابات المحفوظة في شاشة الدخول.
- إضافة ملف README كامل يشرح المشروع، الملفات، التشغيل، Firebase، الصفحات، الصلاحيات، النشر، والملاحظات الأمنية.
- رفع إصدار الكاش في Service Worker إلى `shm-cache-v5` لتقليل مشكلة بقاء نسخة قديمة بعد النشر.

## 4. طريقة التشغيل محلياً

لا تفتح الملف مباشرة عبر `file://` لأن ملفات ES Modules وFirebase قد لا تعمل بشكل صحيح.

استخدم سيرفر محلي بسيط:

```bash
python -m http.server 8000
```

ثم افتح:

```text
http://localhost:8000
```

أو استخدم إضافة Live Server داخل VS Code.

## 5. طريقة النشر

### GitHub Pages

1. ارفع الملفات كلها في جذر المستودع.
2. تأكد أن `index.html` موجود في الجذر.
3. من إعدادات المستودع اختر:
   - `Settings`
   - `Pages`
   - اختر الفرع المناسب مثل `main`
   - اختر `/root`
4. افتح رابط GitHub Pages بعد النشر.
5. عند ظهور نسخة قديمة، نفّذ Hard Refresh أو احذف كاش الموقع من المتصفح.

### Firebase Hosting

يمكن نشره كـ static site. يجب أن يكون `index.html` في مجلد النشر، مع باقي الملفات بجانبه.

### Cloudflare Pages

لا يحتاج build command. اجعل output directory هو الجذر إذا كانت الملفات مرفوعة مباشرة.

## 6. Firebase

### نوع قاعدة البيانات

المشروع يستخدم:

```text
Firebase Realtime Database
```

ولا يستخدم Firestore في النسخة الحالية.

### ملف الإعداد

إعداد Firebase موجود في:

```text
firebase-config.js
```

المفاتيح الظاهرة في تطبيقات Firebase Web ليست سرّية بحد ذاتها، لكن الحماية الحقيقية يجب أن تكون من خلال Rules مناسبة على قاعدة البيانات.

### مسارات البيانات المستخدمة

| المسار | الاستخدام |
|---|---|
| `jawaher_orders` | الطلبات. |

| `jawaher_returns` | المرتجعات. |
| `jawaher_purchases` | المشتريات. |
| `jawaher_def/pages` | تعريف صفحات البيع/المصادر. |
| `jawaher_def/entryUsers` | تعريف مستخدمي الإدخال/المندوبين الظاهرين في الطلبات. |

| `jawaher_transfers` | تحويلات مستودع المؤسسة. |

## 7. ملاحظة مهمة عن Firebase Rules

النظام الحالي يستخدم تسجيل دخول داخلي داخل التطبيق، وليس Firebase Authentication كامل. لذلك، إذا تم ضبط Realtime Database Rules على `auth != null` فقط، فلن تعمل القراءة والكتابة إلا بعد إضافة Firebase Auth فعلياً.

لبيئة الإنتاج، الأفضل تقنياً هو:

1. تفعيل Firebase Authentication.
2. ربط كل مستخدم نظام بحساب Firebase Auth.
3. نقل التحقق من الصلاحيات إلى Firebase Rules قدر الإمكان.
4. منع الكتابة المباشرة للمستخدمين غير المصرح لهم.

أما في النسخة الحالية، فالصلاحيات مطبقة على مستوى واجهة التطبيق والمنطق الداخلي، لذلك يجب التعامل مع قواعد Firebase بحذر وعدم اعتبار الواجهة وحدها طبقة أمان كافية ضد مستخدم تقني قادر على الوصول المباشر لقاعدة البيانات.

## 9. تذكّر كلمة المرور والحسابات المحفوظة

تم إصلاح ميزة التذكر في هذه النسخة.

### السلوك الحالي

- عند تفعيل خيار `تذكّرني` وتسجيل الدخول بنجاح، يتم حفظ اسم المستخدم وكلمة المرور محلياً في المتصفح.
- تظهر الحسابات المحفوظة في شاشة الدخول.
- عند الضغط على حساب محفوظ يحتوي كلمة مرور، يتم الدخول السريع مباشرة.
- إذا كان الحساب قديماً ومحفوظاً بدون كلمة مرور، سيطلب النظام إدخال كلمة المرور.
- يمكن حذف الحساب المحفوظ من زر `×` بجانب الحساب.
- يتم حفظ آخر مستخدم في المفتاح المحلي `shmRememberLastUser`.

### التخزين المحلي

يتم استخدام:

```text
localStorage.shmSavedAccounts
localStorage.shmRememberLastUser
```

كلمة المرور مخزنة بصيغة مموّهة Base64 وليست تشفيراً أمنياً قوياً. هذا مناسب كتسهيل تشغيل داخلي على جهاز موثوق، وليس بديلاً عن Firebase Authentication أو مدير كلمات مرور حقيقي.

## 10. التنقل والـ FAB

### زر FAB

الزر العائم يظهر عند التمرير أو عندما تكون الصفحة طويلة، بشرط:

- أن يكون المستخدم مسجل الدخول.
- أن تكون صفحة `طلب جديد` متاحة له.
- ألا يكون موجوداً أصلاً داخل صفحة `طلب جديد`.

وظيفته:

```text
الانتقال مباشرة إلى صفحة إدخال طلب جديد
```

مع تمرير ناعم إلى كارد الإدخال.

### إخفاء وإظهار التنقل

عند التمرير للأسفل، يقوم النظام بإخفاء شريط التنقل لتوفير مساحة خاصة على الموبايل. عند حدوث ذلك يظهر زر صغير باسم:

```text
القائمة
```

وظيفته إعادة إظهار التنقل فوراً.

## 22. ملاحظات أمنية

- لا تعتبر `localStorage` مكاناً آمناً لكلمات المرور الحساسة.
- ميزة `تذكّرني` في هذه النسخة مخصصة لتسهيل الاستخدام على أجهزة موثوقة فقط.
- إذا كان النظام سيستخدم من عدة موظفين وعلى أجهزة مشتركة، الأفضل تعطيل تذكر كلمة المرور أو الانتقال إلى Firebase Authentication.
- لا تعتمد على إخفاء الأزرار في الواجهة كحماية نهائية. يجب حماية قاعدة البيانات بقواعد مناسبة.
- يجب عدم نشر قواعد Firebase مفتوحة في بيئة إنتاج حقيقية.

## 23. استكشاف الأخطاء

### الموقع يعرض نسخة قديمة بعد رفع التعديلات

الحل:

1. افتح DevTools.
2. من Application > Service Workers اختر Unregister.
3. من Storage اختر Clear site data.
4. أعد تحميل الصفحة.

أو جرّب Hard Refresh:

```text
Ctrl + F5
```

### Firebase لا يقرأ أو لا يكتب

افحص التالي:

- صحة `databaseURL` داخل `firebase-config.js`.
- حالة Realtime Database في Firebase Console.
- قواعد Realtime Database Rules.
- وجود المسارات المطلوبة.
- اتصال الإنترنت.

### الأيقونات لا تظهر

افحص تحميل Font Awesome من CDN. إذا كان الموقع محجوباً عن CDN أو يعمل دون إنترنت لأول مرة، قد لا تظهر الأيقونات.

### Excel لا يعمل

افحص تحميل مكتبة XLSX من CDN:

```text
cdnjs.cloudflare.com/ajax/libs/xlsx/0.18.5/xlsx.full.min.js
```

### الباركود لا يظهر

افحص تحميل مكتبة JsBarcode من CDN:

```text
cdnjs.cloudflare.com/ajax/libs/jsbarcode/3.11.6/JsBarcode.all.min.js
```
