# مؤسسة شادي الملكاوي — نظام إدارة المبيعات والمستودع

نظام ويب تشغيلي لإدارة الطلبات، التوصيل، العملاء، المخزون، المشتريات، المرتجعات، تحويلات المخزون، المستخدمين، الصلاحيات، التقارير، وسجل التدقيق المالي. المشروع مبني كتطبيق صفحة واحدة `Single Page Application` باستخدام `HTML + CSS + JavaScript ES Modules` مع `Firebase Realtime Database`، ولا يحتاج إلى Build أو Node.js للتشغيل.

---

## 1. معلومات النسخة

| البند | القيمة |
|---|---|
| اسم النظام | مؤسسة شادي الملكاوي — إدارة المبيعات والمستودع |
| نوع المشروع | Static Web App / Single Page App |
| الواجهة | HTML, CSS, Vanilla JavaScript |
| قاعدة البيانات | Firebase Realtime Database |
| نقطة التشغيل | `index.html` |
| ملف المنطق الرئيسي | `app.js` |
| ملفات التصميم | `style.css`, `style-shm.css` |
| Service Worker | `sw.js` |
| رقم النسخة | `2026.06.17-ui-contrast-remember-readme` |
| Cache Name | `shm-cache-v6` |

## 2. آخر التعديلات في هذه النسخة

- إصلاح تباين عناوين جميع الجداول، خصوصاً صفحات العملاء، المستخدمين، سجل التدقيق، والتقارير.
- توحيد شكل رأس الجدول بخلفية داكنة ونص أبيض عالي التباين.
- تحسين قابلية قراءة خلايا الجدول وإضافة تمييز خفيف للصفوف المتبادلة والصف عند المرور.
- إصلاح ظهور اسم المستخدم المحفوظ في شاشة الدخول؛ كان النص غير واضح بسبب استخدام لون أبيض على بطاقة فاتحة.
- تحسين تصميم الحسابات المحفوظة في شاشة الدخول بشكل كامل.
- إضافة نص واضح بجانب خيار تذكر كلمة المرور: `تذكّرني`.
- إصلاح سلوك استرجاع الحساب المحفوظ بعد تسجيل الخروج أو إعادة فتح شاشة الدخول.
- إبقاء زر `FAB` عالي التباين فوق الخلفيات الفاتحة والجداول.
- تحسين زر إظهار القائمة عند إخفاء شريط التنقل أثناء التمرير.
- تحديث `version.json` ورفع إصدار Service Worker Cache.

## 5. طريقة التشغيل محلياً

لا يفضّل تشغيل المشروع مباشرة عبر `file://` لأن `ES Modules` وFirebase قد لا تعمل بشكل صحيح.

شغّل سيرفر محلي من داخل مجلد المشروع:

```bash
python -m http.server 8000
```

ثم افتح:

```text
http://localhost:8000
```

أو باستخدام VS Code:

1. افتح مجلد المشروع.
2. ثبّت إضافة Live Server.
3. افتح `index.html`.
4. اضغط `Open with Live Server`.

## 9. ميزة تذكّرني والحسابات المحفوظة

ميزة `تذكّرني` تحفظ آخر حساب تم استخدامه محلياً داخل المتصفح.

ما الذي يتم حفظه؟

- اسم المستخدم.
- كلمة المرور بصيغة مموّهة `Base64` محلياً.
- آخر حساب مستخدم.

السلوك الحالي:

- عند تفعيل `تذكّرني` ثم تسجيل الدخول، يظهر الحساب لاحقاً ضمن الحسابات المحفوظة.
- عند الضغط على الحساب المحفوظ، يتم تعبئة البيانات وتنفيذ دخول سريع إذا كانت كلمة المرور محفوظة.
- بعد تسجيل الخروج، يتم إظهار الحساب المحفوظ وإعادة تعبئة البيانات عند توفرها.
- يمكن حذف الحساب المحفوظ من زر الحذف داخل بطاقة الحساب.

ملاحظة أمنية مهمة:

`Base64` ليس تشفيراً أمنياً حقيقياً. هذه الميزة مناسبة لجهاز موثوق فقط. للحماية الفعلية يجب استخدام Firebase Authentication أو نظام جلسات آمن من جهة الخادم.

## 21. Service Worker والكاش

الملف:

```text
sw.js
```

يستخدم Cache Name:

```text
shm-cache-v6
```

وظيفته:

- حفظ ملفات Shell الأساسية.
- تحميل آخر نسخة من الملفات مع `cache: no-store` للملفات المحلية.
- حذف الكاش القديم عند تفعيل نسخة جديدة.
- دعم فتح `index.html` عند عدم توفر الإنترنت جزئياً.

بعد رفع نسخة جديدة، إذا بقي المتصفح يعرض نسخة قديمة:

1. افتح DevTools.
2. Application.
3. Service Workers.
4. اضغط Unregister.
5. امسح Cache Storage.
6. أعد تحميل الصفحة Hard Refresh.

- الحسابات المدمجة مناسبة للتشغيل السريع أو النسخ الداخلية، لكنها ليست نموذج أمان مثالي.

- الأفضل مستقبلاً الانتقال إلى Firebase Authentication مع Custom Claims أو صلاحيات من قاعدة البيانات.
- يجب ضبط Firebase Database Rules بما يمنع القراءة والكتابة العامة غير المصرح بها.

## 25. فحص الأعطال الشائعة

### الصفحة لا تعمل عند فتح `index.html` مباشرة

السبب غالباً أن المتصفح يمنع ES Modules عبر `file://`.

الحل:

```bash
python -m http.server 8000
```

### التعديلات لا تظهر بعد رفع النسخة

السبب غالباً Service Worker أو Cache.

الحل:

- Hard Refresh.
- حذف Cache Storage.
- Unregister للـ Service Worker.
- التأكد من أن `CACHE_NAME` تغير في `sw.js`.

### بيانات Firebase لا تظهر

افحص:

- صحة `databaseURL`.
- قواعد Realtime Database.
- اتصال الإنترنت.
- Console في المتصفح.
- أسماء المسارات في `firebase-config.js`.

### أزرار Excel لا تعمل

افحص تحميل مكتبة:

```text
xlsx.full.min.js
```

إذا لم تكن المكتبة محملة بسبب الإنترنت أو CDN، لن يعمل التصدير.

### الباركود لا يظهر

افحص تحميل مكتبة:

```text
JsBarcode.all.min.js
```

## 26. إرشادات التعديل

عند تعديل المشروع:

1. لا تغيّر أسماء `id` داخل `index.html` دون تعديل `app.js`.
2. لا تغيّر أسماء مسارات Firebase دون تعديل `firebase-config.js` و`app.js`.
3. أي تعديل على التصميم العام يفضّل وضعه في نهاية `style-shm.css` حتى يتغلب على القواعد السابقة.
4. بعد أي تعديل مهم، غيّر رقم النسخة في `version.json` واسم الكاش في `sw.js`.
5. افحص Console بعد كل تعديل.

## 29. ملخص تشغيلي سريع

لتشغيل النظام:

1. افتح المجلد بسيرفر محلي.
2. تأكد من إعداد `firebase-config.js`.
3. افتح `index.html` من المتصفح.
4. سجّل الدخول بحساب صحيح.
5. استخدم صفحات الطلبات، التقارير، المخزون، والعملاء حسب الصلاحيات.

لتحديث نسخة منشورة:

1. عدّل الملفات.
2. غيّر `version.json`.
3. غيّر `CACHE_NAME` في `sw.js`.
4. ارفع الملفات.
5. امسح كاش المتصفح إذا لزم.
