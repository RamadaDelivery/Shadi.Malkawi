# مؤسسة شادي الملكاوي — نظام إدارة المبيعات والمستودع

نظام ويب عربي لإدارة الطلبات، العملاء، المخزون، المشتريات، المرتجعات، التقارير، المستخدمين، وسجل العمليات. يعمل بواجهة واحدة مبنية بـ HTML/CSS/Vanilla JavaScript مع Firebase Realtime Database، بدون React وبدون Build Step.

---

## الإصدار الحالي

```json
2026.06.17-v8-stable-nav-context-fab-warehouse-collapse
```

كاش Service Worker الحالي:

```js
shm-cache-v8
```

---

## أهم التعديلات الأخيرة

- تثبيت شريط التنقل ومنع اختفائه أثناء التمرير في كل الصفحات.
- تحويل زر FAB إلى أزرار إجراءات سريعة مرتبطة بالصفحة الحالية.
- تحسين FAB ليعرض إجراءات مختلفة حسب السياق: طلب جديد، تصدير، مسح باركود، مشتريات، مستخدم جديد، إلخ.
- تحسين بطاقات المستودع بحيث تعرض أول 5 مقاسات/ألوان فقط.
- إضافة زر عرض/إخفاء لباقي المقاسات والألوان داخل بطاقة الصنف.
- إعادة تصميم صفوف المقاس/اللون/الباركود داخل بطاقة المستودع لتكون أوضح على الموبايل.
- الإبقاء على ملف تصميم واحد فقط: `style.css`.
- الإبقاء على صفحة `إعدادات الموقع` بدلاً من فصل التعريفات والمستخدمين.
- الإبقاء على صفحة `السجل والتدقيق` كصفحة موحدة في آخر شريط التنقل.

## بنية الملفات

| الملف | الوصف |
|---|---|
| `index.html` | هيكل الواجهة الكامل، الصفحات، شاشات الدخول، النوافذ، التنقل، وأزرار FAB. |
| `app.js` | منطق التطبيق بالكامل: الدخول، الطلبات، المخزون، التقارير، العملاء، المستخدمون، السجلات، Firebase. |

| `version.json` | رقم إصدار النشر الحالي. |

## الصفحات الأساسية

### لوحة القيادة
تعرض مؤشرات تشغيلية مختصرة عن الطلبات، الإيرادات، المخزون، والتنبيهات.

### الطلبات
بورد متابعة الطلبات حسب الحالة، مع عرض تفاصيل الطلب، واتساب، الطباعة، تعديل الحالة، والحذف حسب الصلاحية.

### طلب جديد
معالج إدخال طلب بخطوات متتابعة، يشمل بيانات العميل، المحافظة، العنوان، قناة التواصل، المنتجات، اللون، المقاس، الكمية، السعر، والصفحة/المصدر.

### التقارير
بحث وتصفية للطلبات مع دعم التصدير إلى Excel حسب الصلاحيات.

### المستودع
إدارة المنتجات والكميات والباركود. بطاقات المستودع تعرض أول 5 مقاسات/ألوان للصنف، مع زر عرض/إخفاء لباقي التفاصيل عند الحاجة.

### المشتريات
تسجيل دخول بضاعة جديدة أو تحديث منتج موجود، مع دعم الباركود، المقاسات، الألوان، الأسعار، والصفحة/المصدر.

### المرتجعات
تسجيل مرتجع وربطه بالمخزون حتى ترجع الكمية بشكل منطقي إلى المستودع.

### العملاء
CRM مبسط يعرض العملاء، أرقامهم، المحافظات، عدد الطلبات، الإيرادات، متوسط الطلب، آخر طلب، والتصنيف.

### إعدادات الموقع
صفحة موحدة تحتوي:

- الصفحات/المصادر.
- المستخدمين.
- الصلاحيات.
- الألوان المخصصة.
- إنشاء وتعديل وتعطيل المستخدمين.

### السجل والتدقيق
صفحة موحدة تجمع سجل العمليات وسجل التدقيق في جدول واحد، وهي آخر صفحة في شريط التنقل.

## أزرار FAB السريعة

زر FAB لم يعد زر “طلب جديد” ثابت فقط. أصبح Contextual Quick Actions حسب الصفحة:

| الصفحة | أمثلة على الإجراءات السريعة |
|---|---|
| لوحة القيادة | طلب جديد، الطلبات، التقارير |
| الطلبات | طلب جديد، التقارير، تصدير Excel |
| طلب جديد | التالي/حفظ، الطلبات، طباعة آخر طلب |
| التقارير | تصدير Excel، طلب جديد، بحث سريع |
| المستودع | منتج جديد، مسح باركود، مشتريات |
| المشتريات | مسح باركود، حفظ الشراء، المستودع |
| المرتجعات | مسح مرتجع، الطلبات، طلب جديد |
| العملاء | تصدير العملاء، التقارير، طلب جديد |
| إعدادات الموقع | مستخدم جديد، لون مخصص، السجل |
| السجل والتدقيق | لوحة القيادة، التقارير، طلب جديد |
| حركة الصنف | تصدير الحركة، المستودع |

الأزرار تحترم الصلاحيات قدر الإمكان، ولا تظهر الإجراءات الإدارية للمستخدم غير المصرح.

## المستودع والمقاسات والألوان

منطق عرض الصنف داخل المستودع:

- يتم حساب إجمالي المخزون من كل المقاسات/الألوان.
- يتم عرض أول 5 Variants فقط داخل بطاقة الصنف.
- عند وجود أكثر من 5 Variants يظهر زر:
  - عرض المقاسات/الألوان الإضافية.
  - إخفاء التفاصيل.
- كل Variant يعرض:
  - المقاس.
  - اللون مع نقطة لونية.
  - الكمية.
  - الباركود.
  - زر تعديل الباركود.

كل Variant يجب أن يبقى له باركود مستقل عند توفره داخل `variations`.

## نظام الدخول وتذكّرني

شاشة الدخول تدعم:

- اسم المستخدم.
- كلمة المرور.
- خيار `تذكّرني`.
- حسابات محفوظة.
- دخول سريع للحسابات المحفوظة.

ملاحظة أمنية: حفظ كلمة المرور محلياً يتم بصيغة مموّهة محلياً، وليس تشفيراً مؤسسياً قوياً. استخدمه فقط على أجهزة موثوقة.

## Firebase

يعتمد النظام على Firebase Realtime Database من خلال `firebase-config.js`.

أهم المسارات المنطقية:

- الطلبات.
- المستودع.
- المشتريات.
- المرتجعات.
- التعريفات.
- المستخدمون.
- الألوان المخصصة.
- السجلات.
- التدقيق.

لا تغيّر أسماء المسارات قبل مراجعة `firebase-config.js` و `app.js` معاً.

## Service Worker والكاش

عند تعديل ملفات رئيسية، يجب رفع رقم الكاش داخل `sw.js` حتى تصل النسخة الجديدة للمستخدمين.

الملفات التي تستدعي تحديث الكاش غالباً:

- `index.html`
- `app.js`
- `style.css`
- `constants.js`
- `firebase-config.js`
- `version.json`

## التصدير

التصدير إلى Excel يعتمد على SheetJS.

أماكن التصدير:

- التقارير.
- العملاء.
- حركة الصنف.
- أي جدول تشغيلي مربوط بصلاحية التصدير.

---

## قواعد تعديل مهمة

1. لا تضف ملف CSS ثاني.
2. لا تغيّر أسماء `id` في HTML دون تعديل كل مراجعها في `app.js`.
3. لا تحذف صفحة أو زر قبل البحث عن استخدامه في JavaScript.
4. لا تغيّر مسارات Firebase دون تحديث المراجع.
5. بعد كل تعديل رئيسي حدّث `version.json` و `CACHE_NAME` في `sw.js`.
6. افحص `app.js` قبل التسليم.
7. لا تضف React أو TypeScript أو Tailwind.
